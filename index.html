Understanding Your Data
Where & How
Your Data is Stored
💾
Current Storage: Browser localStorage
Right now all ticket data is saved in the browser's built-in localStorage on the device where Admin Portal is opened. It works instantly with zero setup — no server needed.
Data stays permanently until browser cache is manually cleared
All tickets visible in Admin Portal on the same browser
Export CSV/PDF anytime to backup your data
Best for: Single-device admin use, quick start
// Where data lives in your browser: localStorage.getItem('jkumar_helpdesk_tickets') // To manually backup — open browser console & run: copy(localStorage.getItem('jkumar_helpdesk_tickets')) // Then paste in Notepad and save as .json file
📤
Export = Your Permanent Backup
Every time you export CSV or PDF from Admin Portal, that becomes your permanent data record.
Export CSV → Open in Excel → Save as .xlsx
Export PDF → Print or Save to Google Drive
GM Report → Print directly for meetings
Do this regularly (weekly/monthly)
☁️
Future Upgrade: Google Sheets Backend
For multi-device access (so any employee on any computer can see tickets), a Google Sheets backend can be added — completely free.
All tickets auto-saved to Google Sheet

⚠️
Important: Don't Lose Your Data
Follow these steps to keep your ticket data safe:
Export CSV every Friday (end of week backup)
Never clear browser cache / history on admin PC
Always use same browser on same PC for admin
Save exported files in company shared folder
